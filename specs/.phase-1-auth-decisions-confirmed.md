### ✅ Decision 2: JWT Token Generation = Supabase Issues It

**Confirmed**: **Supabase Auth generates + signs JWT** (not custom code)

**What This Means**:
```
Login Flow:
  User enters credentials
  ↓
  SupabaseAuthClient calls POST /auth/v1/token
  ↓
  Supabase verifies + returns: { access_token, refresh_token, user }
  ↓
  We extract access_token → store in HttpOnly cookie
  ↓
  On next request: SupabaseUserContext reads token claims
```

**Implementation Simplification**:
```csharp
// REMOVED from spec:
- ❌ JwtTokenService (custom token generation)
- ❌ Custom token signing logic
- ❌ Custom claims configuration

// KEPT in spec:
- ✅ SupabaseJwtTokenProvider (validates tokens from Supabase)
- ✅ SupabaseUserContext (extracts UserId from token)
```

**JWT Token Structure** (issued by Supabase):
```json
{
  "sub": "user-uuid-here",      // UserId (tenant identifier)
  "exp": 1708050456,            // Expiry (1 hour from now)
  "iat": 1708046856,            // Issued at
  "email": "user@example.com",
  "aud": "authenticated"
}
```

**Validation**:
- Supabase signs with HS256 + JWT Secret
- We validate on each request using Supabase's public key
- **No custom signing code needed**

**Benefits**:
- 👍 Industry standard approach
- 👍 Proven security practices
- 👍 Refresh tokens included (Phase 2+)
- 👍 Token revocation supported

## Environment Configuration

### Required Variables
```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=<anon-key-from-supabase>
SUPABASE_JWT_SECRET=<jwt-secret-from-supabase-settings>
ALLOWED_ORIGINS=https://localhost:7001,https://example.com
```

### How to Get These:
1. Create Supabase project: supabase.com
2. Go to Settings → API keys
   - Copy "anon" key → SUPABASE_ANON_KEY
   - Copy "Service role secret" → SUPABASE_JWT_SECRET (read-only, don't expose)
3. Copy project URL → SUPABASE_URL
