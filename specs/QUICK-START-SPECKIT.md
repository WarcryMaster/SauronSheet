## 🚀 Phase 0: Right Now

### Step 1: Read the Specification (45-60 min)

```bash
# Open and review Phase 0 specification
cat phase-0-foundation-spec.md

# Or open in editor:
code phase-0-foundation-spec.md
```

**What to focus on:**
- Executive Summary (what we build + why)
- Deliverables section (what files to create)
- Test Specifications (what tests to write first)
- Task Breakdown (phases I-VII)

### Step 2: Generate Tasks File

```bash
# Using speckit.tasks command
speckit.tasks phase-0-foundation-spec.md --output phase-0-foundation-tasks.md
```

**Output**: `phase-0-foundation-tasks.md`
- Contains: Ordered list of all 28 tasks
- Format: Task ID, description, estimate, dependencies
- Use: Reference while implementing

### Step 3: Generate Implementation Guide

```bash
# Using speckit.implement command
speckit.implement phase-0-foundation-tasks.md --output phase-0-foundation-implement.md
```

**Output**: `phase-0-foundation-implement.md`
- Contains: Step-by-step implementation instructions
- Format: Code snippets, file paths, exact steps
- Use: Your development playbook

### Step 4: Start Coding

Open `phase-0-foundation-implement.md` and follow the guide:

```
Phase I: Initialization (Tasks I-1 to I-10)
  └─ I-1: Create solution file (dotnet new sln)
  └─ I-2: Create Domain project
  └─ I-3: Create Application project
  └─ I-4: Create Infrastructure project
  └─ I-5: Create Frontend project
  └─ ... (follow implement.md exactly)

Phase II: Domain Foundation (Tasks II-1 to II-10)
  └─ II-1: Create Entity<TId> base class
  └─ II-2: Create ValueObject base class
  └─ ... (continue as per implement.md)

... (continue through Phase VII)
```

### Step 5: Verify Progress

After each phase, run:

```bash
dotnet build                    # Must show 0 warnings
dotnet test                     # Track passing tests (0 → 11)
git status                      # Should be clean after commits
```

### Step 6: Move to Next Phase (Week 4)

When Phase 0 is complete (all 11 tests passing):

```bash
# Phase 1: Same workflow
speckit.tasks phase-1-auth-spec.md --output phase-1-auth-tasks.md
speckit.implement phase-1-auth-tasks.md --output phase-1-auth-implement.md

# Then follow phase-1-auth-implement.md for Phase 1 implementation
```

## ❓ FAQ

**Q: Do I need to read all 7 specs now?**  
A: No. Read phase-0-foundation-spec.md now. Read phase-1-auth-spec.md when Phase 0 is done.

**Q: What if speckit.tasks/implement fail?**  
A: You can manually create the tasks.md and implement.md files from the spec. (Check speckit docs for CLI syntax)

**Q: Can I skip phases?**  
A: No. Each phase depends on the previous one. Exception: Phase 5 (budgets) is optional.

**Q: How do I know I'm done with a phase?**  
A: All tests pass (`dotnet test` = full count) + `dotnet build` shows 0 warnings.

**Q: Where do I track my progress?**  
A: Use daily standup template in phase-X-foundation-implement.md.
